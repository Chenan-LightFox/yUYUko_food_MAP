# 一、整体系统架构建议

一般建议拆成三层：

```
前端（地图展示 + 交互）
后端（业务逻辑 + 权限 + 数据处理）
数据库（存储地点与评论等数据）
```

### 推荐技术栈（比较适合学生项目+可扩展）

#### 前端

* Vue3 + Vite（简单好上手）
* 或 React（如果你更熟）
* 高德 JS API（核心地图）

#### 后端

* Node.js + Express / NestJS
* 如果想写得规范一点推荐 NestJS

#### 数据库

* MySQL / PostgreSQL（结构化数据）
* 如果评论很多可考虑 MongoDB（可选）

# 二、核心功能模块设计

---

## ⭐ 1. 地点管理模块

这是你网站的核心。

### 需要支持功能

#### ✔ 添加地点

用户可以：

* 在地图点击选点
* 输入地点名称
* 分类（火锅、烧烤、甜品等）
* 上传图片
* 打分
* 写推荐理由

👉 技术实现

* 高德地图点击事件
* 获取经纬度
* 提交到后端

示例流程：

```
用户点击地图
→ 获取经纬度
→ 弹出表单
→ 提交到服务器
→ 存数据库
```

---

#### ✔ 删除地点

通常设计成：

* 仅创建者能删
* 或管理员能删

---

#### ✔ 评论系统

评论建议单独表结构：

```
Comment
- id
- place_id
- user_id
- content
- rating
- time
```

## ⭐ 2. 查询与搜索模块

你这里提的两个搜索非常关键。

---

### ✔ 精确搜索

比如输入：

```
阿明牛肉面
```

实现方式：

#### 方案1（简单）

数据库模糊查询

```sql
SELECT * FROM places
WHERE name LIKE '%牛肉面%'
```

---

#### 方案2（高级）

可以接入：

* ElasticSearch（后期升级）

---

---

### ✔ 范围搜索（地图项目重点）

例如：

👉 搜索当前视野内的餐厅

实现思路：

1. 获取地图可视范围

```
NE（东北角）
SW（西南角）
```

高德 API：

```javascript
map.getBounds()
```

2. 把范围传给后端
3. 数据库进行地理范围查询

例如：

```sql
SELECT *
FROM places
WHERE latitude BETWEEN ? AND ?
AND longitude BETWEEN ? AND ?
```

# 四、地图前端关键实现点

---

## 📍 加载地点标记

流程：

```
页面加载
→ 请求后端所有地点
→ 循环创建 Marker
```

高德示例：

```javascript
const marker = new AMap.Marker({
  position: [lng, lat],
  title: place.name
});
marker.setMap(map);
```
